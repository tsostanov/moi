# Лабораторная работа №5 — план

## Тема
Снижение шума на синтезированном изображении методом билатеральной фильтрации
с сохранением границ объектов и физической корректности.

## Постановка (из task.pdf, слайды 44–45)
- **Цель:** снизить шумы на синтезированном изображении, сохранив физическую
  корректность.
- **Входные данные:** изображение со следующей информацией в точке первого
  пересечения луча со сценой (AOV-каналы):
  - Прямая яркость (direct radiance)
  - Вторичная яркость (indirect radiance)
  - Глубина (depth)
  - Индекс объекта сцены (object id)
  - Нормаль к поверхности (normal)
- **Задание:** реализовать билатеральную фильтрацию с сохранением границ
  объектов; показать корректность результатов.
- **Веса по поверхности:** арифметическое среднее (проще) или медианный
  фильтр (сложнее).
  - Арифметическое среднее:
    `g_p = (1/W_p) Σ_{q∈S} f_q · G_s(p−q) · G_r(p−q)`,
    при `Σ_{q∈S} G_s(p−q) = 1` достаточно `W_p = Σ_{q∈S} G_r(p−q)`.
  - Медианный фильтр: нормировать всё изображение после фильтрации так,
    чтобы для каждого объекта O:
    `Σ_{p∈O} g_p = Σ_{p∈O} f_q` (сохранение полной энергии на объект).

- Из **04/gui.py** — паттерн запуска через subprocess с логом. По желанию
  добавить кнопки/параметры фильтра в GUI.
- Из **03/main.py** — стиль аккуратной структуры скрипта и сохранения
  серии PNG для сравнения (`*_distribution.png`). Использовать тот же

## Этапы

### 1. Расширение рендера (`05/render_aov.py` или `--dump-aov` в 04/main.py)
1. Добавить в трассировку первичного луча запись AOV в точке первого
   пересечения:
   - `direct[i,j]` — оценка прямого освещения (`estimate_direct_light` без
     умножения на throughput других глубин).
   - `indirect[i,j]` — вторичная яркость (полный путь минус прямой вклад).
   - `depth[i,j]` — `t` ближайшего пересечения первичного луча.
   - `obj_id[i,j]` — индекс объекта (потребуется отнести каждый треугольник
     к объекту: стены, источник, объект 1, объект 2 … Простой способ —
     назначать `object_id` группе треугольников при `add_quad/add_box/...`).
   - `normal[i,j]` — нормаль поверхности (3 канала).
2. Для пикселей-фоновых лучей (без попадания) использовать sentinel
   (`obj_id = -1`, `depth = +inf`, `normal = 0`).
3. Сохранять AOV в один `npz` (`np.savez_compressed`) рядом с `render.png`:
   `05/outputs/aov.npz` с ключами `direct, indirect, depth, obj_id, normal,
   color = direct+indirect`.
4. Для отчёта рендерить заведомо шумный кадр (например, `--samples 1..4`,
   `--max-depth 4`) и более чистый эталон (`--samples 128..256`) — эталон
   нужен для оценки качества фильтрации (PSNR/L1).

### 2. Базовый билатеральный фильтр (`05/bilateral.py`)
1. Реализовать пространственный гаусс `G_s` через предвычисленное ядро
   `kernel_s[dy,dx] = exp(-(dy²+dx²)/(2σ_s²))`, нормированное так, чтобы
   `Σ_{q∈S} G_s(p−q) = 1` — ровно условие со слайда 45. Тогда из общей
   формулы `g_p = (1/W_p) Σ f_q G_s G_r` получаем `W_p = Σ_{q∈S} G_r(p−q)`,
   и нормировку считаем одним проходом по окну вместе с числителем.
   ВАЖНО: нормируется именно дискретное ядро по конечному окну S, а не
   непрерывный гауссиан — иначе сумма не будет равна 1 и условие
   слайда нарушится.
2. Диапазонный вес `G_r` собрать как произведение нескольких edge-stop
   функций по AOV-каналам:
   - `w_obj = 1 if obj_id_p == obj_id_q else 0` — жёсткий запрет смешивать
     разные объекты, гарантирует сохранение границ объектов.
   - `w_normal = exp(-(1 − max(0, n_p·n_q)) / σ_n²)` — сохраняет углы.
   - `w_depth  = exp(-(z_p − z_q)² / (2σ_z²))` — отделяет
     перекрывающиеся объекты, лежащие в одном `obj_id`.
   - (опц.) `w_color = exp(-||c_p − c_q||² / (2σ_c²))` по самому шумному
     каналу: применять с осторожностью, иначе фильтр «приваривает» шум.
3. Фильтровать **отдельно** `direct` и `indirect`. Это соответствует
   физическому смыслу: компоненты независимы и имеют разную дисперсию.
   Итог: `filtered = filtered_direct + filtered_indirect`.
4. Векторизация: основной цикл по сдвигам `(dy,dx) ∈ window`, внутри —
   numpy-операции над всеми пикселями. Это быстро и читаемо.
5. Граница изображения: `np.pad(..., mode='edge')`.

### 3. Median-вариант (опционально, на «сложнее»)
1. В пределах окна S для каждого пикселя `p` собрать множество соседей `q`
   с одинаковым `obj_id` и (опц.) близкой нормалью, взять покомпонентный
   медианный отсчёт `f_q`.
2. Постнормировка по объектам: для каждого `O` посчитать
   `S_in = Σ_{p∈O} f(q)`, `S_out = Σ_{p∈O} g(p)` и умножить `g|_O` на
   `S_in / S_out`. После этого выполняется ровно условие со слайда 45:
   `Σ_{p∈O} g(p) = Σ_{p∈O} f(q)` для каждого объекта `O`. Применять к
   каждому RGB-каналу независимо. Если `S_out == 0` (объект полностью
   занулился — практически не встречается), оставлять `g|_O` как есть.
3. Тот же приём (постнормировку по объектам) применить и к
   арифметическому среднему — в отчёте показать, что без неё суммарная
   энергия объекта может «уплыть».

### 4. CLI и интерфейс (`05/main.py`)
- `python 05/main.py --aov 05/outputs/aov.npz --output 05/outputs/filtered.png`
- Параметры: `--sigma-s`, `--sigma-n`, `--sigma-z`, `--sigma-c`, `--radius`,
  `--mode {mean,median}`, `--split-direct-indirect`,
  `--energy-normalize {none,object}`, `--reference reference.npz`
  (для PSNR/L1).
- На выходе: `filtered.png`, `filtered.ppm` (как в 04 — переиспользовать
  `write_ppm` / `build_display_bytes`), `filtered.txt` с параметрами и
  метриками.
- По желанию: визуализации AOV для отчёта (`obj_id` в палитре, `normal`
  как (n+1)/2 в RGB, `depth` нормированный) — отдельная команда
  `--dump-debug`.

### 5. Демонстрация корректности (что в отчёт)
1. Сравнение шумного и отфильтрованного изображения бок о бок + diff
   (`|noisy − filtered|`, увеличенный для наглядности).
2. Сравнение с «эталоном» (high-spp рендер): PSNR и средняя L1-ошибка
   для `noisy` и для `filtered` — фильтр должен снижать ошибку.
3. Проверка сохранения границ: специально подобрать кроп границы двух
   объектов и показать, что фильтр без `obj_id`-веса размывает границу,
   а с весом — нет.
4. Проверка энергетической нормировки: вывести `Σ pixels` на каждом
   объекте до/после фильтра. Для `--energy-normalize object` сумма
   совпадает с точностью до численной погрешности.
5. Ablation: фильтр без отдельной обработки direct/indirect и с ней —
   показать разницу в шуме теней.

### 6. Юнит-тесты (`05/tests/`)
Использовать `pytest`. Тесты должны быть быстрыми (синтетические массивы
20×20–50×50, не запускать рендер).

1. `test_spatial_kernel.py`:
   - ядро симметрично, `Σ = 1`, центральный вес максимален.
2. `test_bilateral_preserves_constant.py`:
   - на константном изображении любой фильтр возвращает то же
     изображение (для `mean` и `median`).
3. `test_object_edge_preservation.py`:
   - две полуплоскости с разными `obj_id` и разными цветами; после
     фильтра пиксели одной полуплоскости не получают вклад от другой
     (равенство значений на «своей» стороне с точностью до шума).
4. `test_normal_edge_preservation.py`:
   - то же, но без `obj_id`, а с двумя плоскостями с разными нормалями.
5. `test_energy_conservation.py`:
   - после `--energy-normalize object` для каждого `obj_id` суммарная
     энергия отфильтрованного изображения равна суммарной энергии
     исходного (с tol `1e-6`).
6. `test_noise_reduction.py`:
   - синтетика: гладкий сигнал + гауссов шум, проверить что L1-ошибка
     к эталону у `filtered` меньше, чем у `noisy` (для одного фиксированного
     набора σ).
7. `test_aov_io.py`:
   - круговой тест записи/чтения `aov.npz`: все ключи на месте,
     dtype и shape сохраняются.
8. `test_cli_smoke.py`:
   - запуск `main.py` на мини-AOV (10×10), проверить что файлы
     `filtered.png/.ppm/.txt` создаются и непустые.

Сделать `pytest 05/tests` зелёным до сдачи.

## Риски и на что обратить внимание
- **Object id.** Чтобы он был осмысленным, в 04 нужно сгруппировать
  треугольники: отдельный id для пола, потолка, каждой стены,
  источника, каждого объекта. Хранить рядом с `Triangle` либо в
  параллельном массиве `tri_object_id[N]`. Без этого жёсткое edge-stop
  по объекту бесполезно.
- **Шумный direct.** Если в 04 прямое освещение оценивается одним
  теневым лучом, в `direct` будет много нулей в тенях и ярких пятен на
  границе. Фильтр должен корректно это сглаживать — это видно на
  диффузных стенах рядом с тенями объекта.
- **σ-параметры.** Подобрать на одном кадре, в README зафиксировать
  «дефолтные» значения и обосновать в отчёте.
- **Энергосохранение.** Чисто билатеральный mean уже сохраняет
  суммарную яркость локально (за счёт нормировки на `W_p`), но не
  глобально по объекту. Слайд 45 требует именно объектной нормировки —
  явно реализовать и протестировать.
- **Скорость.** Для 500×500 и окна 7×7 numpy-вариант приемлем
  (несколько секунд). C-ускоритель — по желанию, ради аналогии с ЛР4.
